WampServer & XAMPP Configure with MariaDB and MySQL

第一部分补上次的一个问题

1、WampServer 3不支持的硬件格式 FAT3和 exFAT 他只能工作在NTFS的格式硬盘上。 不能在Windows XP上运行。

        安装 WampServer 必须匹配相应的 Microsoft C/C++ runtime Libraries库 具体的库文件要求如下：

   --- Microsoft Visual C/C++ Redistributable 2008 (VC9),
--- Microsoft Visual C/C++ Redistributable 2010 (VC10),
--- Microsoft Visual C/C++ Redistributable 2012 (VC11),
--- Microsoft Visual C/C++ Redistributable 2013 (VC13),
--- Microsoft Visual C/C++ Redistributable 2015 (VC14),
--- Microsoft Visual C/C++ Redistributable 2017 (VC15)

 说真的现在越来越不喜欢用Windows系统了  ，你会发现各种奇葩的问题。另外一点就是 安装的时候使用根目录最好。关闭之前已经安装的服务（WampServer上有的服务）不然会引发端口占用等问题。还有注册表中的MySQL位置也要修改（之前的博客上面已写过）

第二部分 MariaDB 数据库密码更新问题

     涉及XAMPP中的phpMyAdmin （问题描述： 修改了MySQL数据库中的root密码 之后 phpmyadmin不能进去 ，使用mysql语句重置密码为空 提示没有权限）

1、之前修改了XAMPP中的MySQL 数据库中root的密码，发现 XAMPP中的 phpmyadmin进不去了，这是phpmyadmin中的MySQL账户密码不是同步的，但是XAMPP是没有登陆界面的，所以直接会弹出拒绝登陆界面，但是使用MySQL语句来重新重置密码为空时，MariaDB提示没有权限，

2、MariaDB 数据库更新密码的语句：   update mysql.user set authentication_sting=password('') where user='root' ; 这样就可以将MySQL的root密码重置为空。之后使用刷新 flush privileges ; 退出之后课可以进行测试。 然后在进phpmyadmin 。  之前一直使用的 WampServer 他的phpmyadmin的页面使用登录界面的。

不确定是什么数据可的时候使用version语句查询版本信息  select version() ;   XAMPP的开发者在最新更新的版本上都使用了 MAriaDB。 所以以后XAMPP上不能在部署DVWA了。DVWA只支持MySQL数据库
